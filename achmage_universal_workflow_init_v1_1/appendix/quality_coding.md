# appendix/quality_coding.md — Quality Gates for Coding (v1.1)

목표: 코드 산출물의 동작/가독성/테스트/안전성을 균형 있게 확보하기.

## A) Quick Gate (5분)
- [ ] 실행 방법이 명확하다(README 또는 WORKPLAN에).
- [ ] 핵심 기능이 최소 1회 end-to-end로 동작한다.
- [ ] 에러 처리(입력 검증/예외)가 최소한 있다.
- [ ] 파일 구조가 예측 가능하다(src/, tests/ 등).
- [ ] 다음 작업(버그/리팩터/기능)이 리스트업되어 있다.

## B) Deep Gate (15~40분)
- [ ] 함수/모듈 책임이 과도하게 크지 않다(한 파일에 다 때려넣지 않기).
- [ ] 테스트가 최소 핵심 경로를 커버한다(요구사항 기반).
- [ ] 사이드 이펙트/상태 변화가 명시적이다.
- [ ] 설정과 비밀정보가 분리되어 있다(.env 등, 값은 저장 X).
- [ ] 의존성/버전/환경이 기록되어 있다(가능한 범위에서).

## C) 보안/안전 체크(필수)
- [ ] 외부 요청/다운로드/쉘 실행은 사용자 승인 후에만 한다.
- [ ] 사용자 입력이 파일/명령 실행으로 직접 연결되지 않는다.
- [ ] 민감정보를 로그에 남기지 않는다.

## D) 권장 프롬프트(검토용)
- “요구사항과 WORKPLAN 기준으로 이 코드를 리뷰하고, 버그 가능성/테스트 부족/보안 위험을 체크리스트로 내줘.”
